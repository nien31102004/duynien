Lập Trình Hướng Đối Tượng: Class Thật (Java) và Prototype (JavaScript)

tháng 10 12, 2025
Java • JavaScript • OOP • Cơ Bản

🧠 Giới thiệu Link to heading

Lập trình hướng đối tượng (OOP) là trụ cột của phát triển phần mềm hiện đại. Cả Java và JavaScript đều hỗ trợ OOP, nhưng theo hai triết lý hoàn toàn khác nhau: Java là ngôn ngữ hướng Class, trong khi JavaScript là ngôn ngữ hướng Prototype. Hiểu rõ sự khác biệt này là chìa khóa để viết code OOP hiệu quả trong cả hai môi trường.

⚙️ Phần 1: OOP Thuần Túy trong Java (Class-Based) Link to heading

Java là một ngôn ngữ OOP cổ điển. Mọi thứ trong Java (trừ các kiểu nguyên thủy) đều xoay quanh Class (Lớp).

Class và Kế thừa Link to heading

  • Class là bản thiết kế để tạo ra các Object (Đối tượng).
  • Kế thừa được thực hiện bằng từ khóa extends và implements.
// Java: Class là bản thiết kế
class Animal {
    public void eat() {
        System.out.println("Animal is eating...");
    }
}

// Kế thừa Class
class Dog extends Animal {
    public void bark() {
        System.out.println("Woof!");
    }
}

// Khởi tạo Object
Dog myDog = new Dog();
myDog.eat(); // Output: Animal is eating...

Tóm lại: Java tạo ra các Object từ các Class. OOP của Java dựa trên cấu trúc phân cấp (Inheritance Hierarchy).

🧩 Phần 2: OOP Dựa trên Prototype trong JavaScript Link to heading

Trước ES6 (2015), JavaScript không có từ khóa class. Nó dùng Prototype Chain (Chuỗi Prototype) để thực hiện kế thừa và chia sẻ thuộc tính/phương thức. Mọi Object trong JavaScript đều có một prototype (một Object khác) mà nó có thể kế thừa thuộc tính/phương thức.

Khi bạn truy cập một thuộc tính của Object, nếu Object đó không có, JavaScript sẽ tìm kiếm tiếp trên prototype của nó — cứ thế tạo thành một chuỗi.

Ví dụ: tạo đối tượng và kế thừa qua prototype Link to heading

// JavaScript: Object kế thừa trực tiếp từ Object khác
const animal = {
  eat: function() {
    console.log("Animal is eating...");
  }
};

// Object 'dog' được tạo ra và kế thừa từ 'animal'
const dog = Object.create(animal);
dog.bark = function() {
  console.log("Woof!");
};

dog.eat(); // Output: Animal is eating...

Class “Giả lập” (sugar syntax) trong ES6 Link to heading

ES6 thêm cú pháp class nhưng về bản chất vẫn dùng prototype chain bên dưới.

// JavaScript ES6: Dùng cú pháp Class (thực chất vẫn là Prototype)
class Animal {
  eat() {
    console.log("Animal is eating...");
  }
}

class Dog extends Animal {
  bark() {
    console.log("Woof!");
  }
}

const myDog = new Dog();
myDog.eat(); // Vẫn chạy, vì Dog kế thừa Animal qua Prototype Chain

Nếu muốn viết code JS theo phong cách “chuẩn Java”, hãy dùng cú pháp class/extends của ES6 — nhưng đừng quên rằng bên dưới vẫn là cơ chế prototype hoạt động.
