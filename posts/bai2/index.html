Kiểu Dữ Liệu: Static (Java) vs Dynamic (JavaScript) và Cuộc chiến Var, Let, Const

tháng 10 11, 2025
Java • JavaScript • Kiểu Dữ Liệu • Cơ Bản

🧠 Giới thiệu Link to heading

Hiểu cách một ngôn ngữ quản lý biến và kiểu dữ liệu là bước đầu để viết code hiệu quả và tránh lỗi vặt.
Sự khác biệt lớn nhất giữa Java và JavaScript nằm ở hệ thống kiểu (Type System): Java là kiểu tĩnh, JavaScript là kiểu động. Điều này ảnh hưởng tới cách khai báo, sử dụng và debug code.

⚙️ Cơ chế hoạt động (Runtime) Link to heading

  • Java: code được biên dịch thành bytecode và chạy trên JVM.
  • JavaScript: code được thông dịch / JIT bởi engine (V8, SpiderMonkey, v.v.), chạy trong trình duyệt hoặc Node.js.

Ví dụ Java (biên dịch, kiểu tĩnh) Link to heading

// Java: phải khai báo kiểu
int age = 30;
String name = "Alice";

// Lỗi biên dịch nếu gán sai kiểu:
// age = "Bob"; // Compile-time error
final double PI = 3.14159; // hằng

Ví dụ JavaScript (kiểu động) Link to heading

// JavaScript: kiểu được xác định lúc runtime
let value = 30;     // Number
value = "Thirty";   // bây giờ là String, hợp lệ

const APP_NAME = "MyApp"; // hằng (tham chiếu không thể gán lại)
const settings = { theme: 'dark' };
settings.theme = 'light'; // hợp lệ (thay đổi thuộc tính)

🧩 Var vs Let vs Const (JavaScript) Link to heading

  • var: function scope, dễ gây lỗi do hoisting — hạn chế dùng.
  • let: block scope, dùng cho biến có thể thay đổi.
  • const: block scope, dùng cho hằng (không thể gán lại biến), nhưng nội dung object/array có thể thay đổi.

✅ Kết luận Link to heading

  • Nếu bạn đang chuyển từ Java sang JavaScript: chú ý kiểu động — ưu tiên dùng let/const thay vì var.
  • Nếu bạn đang chuyển từ Java sang JavaScript: nhớ rằng Java yêu cầu khai báo kiểu tĩnh và có lợi cho việc phát hiện lỗi sớm.
